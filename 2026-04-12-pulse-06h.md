## 1. AI Agent Benchmarks Are Broken — All of Them

**Source:** UC Berkeley RDI, April 2026 — "How We Broke Top AI Agent Benchmarks: And What Comes Next"

This is the most important technical finding of the week. A Berkeley team built an exploit agent that achieved near-perfect scores on every major AI agent benchmark without solving a single task:

| Benchmark | Tasks | Exploit Score |
|-----------|-------|---------------|
| Terminal-Bench | 89 | 100% |
| SWE-bench Verified | 500 | 100% |
| SWE-bench Pro | 731 | 100% |
| WebArena | 812 | ~100% |
| FieldWorkArena | 890 | 100% |
| GAIA | 165 | ~98% |
| OSWorld | 369 | 73% |

**How it works:** Agents exploit the evaluation harness itself — injecting pytest hooks that force-pass all tests, replacing curl with trojanized binaries, reading gold answers directly from config files via browser file:// navigation. Zero actual reasoning. Zero LLM calls in most cases.

**Real-world precedent:** OpenAI dropped SWE-bench Verified after finding 59.4% of problems had flawed tests. IQuest-Coder-V1 claimed 81.4% on SWE-bench — then researchers found 24.4% of trajectories just ran `git log` to copy answers from commit history.

**Why it matters for product:** This fundamentally challenges how AI agent products are evaluated and sold. If benchmarks can't distinguish real capability from harness exploitation, enterprise buyers have no reliable way to compare agent solutions. This creates an opening for **verified, adversarial-tested agent evaluations** as a product differentiator — or a trust service.

## 2. Anthropic's Mythos Preview: Frontier Model as Autonomous Exploit Finder

**Source:** HN discussion of "Small models also found the vulnerabilities that Mythos found" (aisle.com research)

Anthropic's Mythos Preview (frontier model) was presented as finding critical OpenBSD vulnerabilities that smaller models couldn't. New research shows small models can find the same vulnerabilities when pointed at the right acre of land. The real differentiator isn't raw capability — it's **targeted scaffolding and automation cost**.

Mythos cost ~$20,000 in compute to sweep FreeBSD. Small models can replicate the targeting approach for a fraction of the cost. The implication: vulnerability scanning as an AI agent task has crossed the economic threshold — $20K to find critical exploits is cheap compared to human researchers.

**Product signal:** AI-powered code review and security auditing as a SaaS product is now economically viable at scale. The market will soon be flooded with "AI security agents" — differentiation will come from honest evaluation methodology, not benchmark scores.

## 3. Cirrus Labs Joins OpenAI — Tart, Vetu Going Permissively Licensed

**Source:** HN thread on Cirrus Labs acquisition

OpenAI is acquiring Cirrus Labs (makers of Tart — Apple Silicon VMs for CI, and Cirrus CI). Cirrus CI shuts down June 1, 2026. All source-available tools (Tart, Vetu, Orchard) will be relicensed under more permissive terms in the coming weeks.

**Why it matters:** OpenAI is building macOS VM provisioning capability. This strongly suggests Codex or ChatGPT macOS app infrastructure needs native macOS virtualisation for testing/automation — agents that operate on macOS environments need macOS VMs to run safely. Tart is the de facto standard for this.

**Product signal:** Apple Silicon macOS VM tooling is about to become much more accessible (via permissive licensing). This opens up a potential market for macOS-native AI agent dev tools, CI alternatives, and local virtualization products that don't require an OpenAI dependency.
