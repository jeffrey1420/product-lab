## 1. Claude Managed Agents — Anthropic's Public Beta Drops April 8

Anthropic launched **Claude Managed Agents** in public beta on April 8, 2026, positioning itself directly against Microsoft Agent Framework and OpenAI's agent infrastructure. This is the most significant Anthropic product launch since Claude 3.5 and represents a fundamental shift in how enterprises consume AI agent infrastructure.

### Architecture: Brain / Hands / Session

The architecture decouples three traditionally entangled components:

- **Session:** Append-only log maintaining agent state across multi-hour tasks. Critical for long-running enterprise workflows.
- **Harness:** The loop that calls Claude and routes tool calls to infrastructure. The "conductor" coordinating agent behavior.
- **Sandbox:** Secure execution environment for code execution, file editing, external service interactions.

This decoupling means each component can evolve independently — change execution logic without touching reasoning, and vice versa. The service acts as a flexible meta-harness, compatible with specific harnesses like Claude Code.

### Agent Teams vs Subagents

Claude Managed Agents offers two coordination modes:

**Agent Teams:** Multiple Claude instances with independent contexts, coordinated by a lead agent via a shared task list (pending/in-progress/completed with dependency management). Teammate agents communicate directly with each other, not just through the lead. Best for complex parallel work.

**Subagents:** Remain in a single session, report only to the main agent. Lower token cost. Best for sequential, well-defined tasks.

| Criterion | Agent Teams | Subagents |
|-----------|-------------|-----------|
| Contexts | Independent per agent | Shared session |
| Communication | Direct between teammates | Report to main agent only |
| Token cost | Higher | Lower |
| Ideal for | Complex parallel work | Sequential targeted tasks |

Recommendation from early reviewers: start with subagents to validate business logic; migrate to Agent Teams only when parallelism is demonstrably beneficial.

### Production Early Adopters

Three companies confirmed in production:

- **Notion:** Custom Agents deployed inside Notion workspaces. Engineers ship code; knowledge workers produce websites and presentations. Dozens of tasks run in parallel while teams collaborate on outputs.
- **Rakuten:** Enterprise agents across product, sales, marketing, finance, and HR deployed within one week per department. Each agent plugs into Slack and Teams, accepts task assignments, returns deliverables.
- **Sentry:** Claude-powered counterpart to their existing Seer debugging agent. Agents write patches and open pull requests automatically.

### Pricing

$0.08 per agent runtime hour, on top of standard Claude token costs. For comparison:
- A 10-hour agent session = $0.80 infrastructure + token costs
- Typical moderately complex task: 200,000-500,000 tokens at Sonnet pricing (~$3/M input, $15/M output)

Claude Code autonomy metric: session length doubled from 25 to 45 minutes over three months — approaching genuinely useful long-task capability.

### Self-Evaluation and Iteration (Research Preview)

Agents can self-evaluate against user-defined success criteria and iterate until met. Built-in session tracing and integration analytics make every tool call, decision, and failure mode inspectable. This is what separates production systems from demos.

**What this means for Alizé:**
- Claude Managed Agents validates the managed agent infrastructure market — if Anthropic is charging $0.08/hour for managed infra, there's clearly demand
- Alizé's positioning can occupy the space above "raw API" but below "full managed service like Managed Agents" — i.e., Alizé handles deployment design, workflow architecture, and governance; the client or Alizé on their behalf uses the managed infrastructure layer
- The Agent Teams vs Subagents model is directly relevant to Alizé's multi-agent orchestration design — Alizé should document when to use each pattern
- Rakuten deploying 5 agents across 5 departments in one week validates that agent deployment, when done right, can be rapid
- Self-evaluation is the missing piece in most enterprise agent deployments — Alizé should make this a standard deliverable in its deployment methodology

## 3. OpenClaude — Open-Source Claude Code Alternative Supporting 200+ Models

Released April 1, 2026, **OpenClaude** is an open-source coding-agent CLI that competes directly with Claude Code while offering broader model support:

- **Supported providers:** OpenAI (and OpenRouter, DeepSeek, Groq, Mistral, LM Studio via OpenAI-compatible API), Google Gemini, GitHub Models, Codex, Ollama (local), Atomic Chat (Apple Silicon)
- **Total model coverage:** 200+ models via OpenAI-compatible endpoints
- **Agent routing:** Per-agent routing to different models via `settings.json` — e.g., "Explore" agent routes to DeepSeek, "Plan" agent routes to GPT-4o
- **Installation:** `npm install -g @gitlawb/openclaude`

The agent routing feature is notable: it enables a single CLI session to use different models for different sub-tasks based on cost/quality tradeoffs. A "general-purpose" agent uses GPT-4o; an "Explore" agent uses cheaper DeepSeek. This multi-model routing within a single workflow is exactly the kind of pattern Alizé should consider for its architecture.

**What this means for Alizé:**
- OpenClaude demonstrates that the "Claude Code CLI" pattern is becoming commoditized — the differentiation is no longer the interface but the backend intelligence and workflow design
- The 200+ model support means Alizé can design workflows that route to the most cost-effective model for each task type — not a single model for everything
- Agent routing at the CLI level prefigures where enterprise agent architecture is heading: dynamic model selection based on task requirements
- Alizé should evaluate OpenClaude as a prototyping tool for testing multi-model workflows before committing to a production architecture

**What this means for Alizé:**

## Product Direction Recommendations

1. **Position Alizé between raw API and managed platforms:** Claude Managed Agents charges $0.08/hour for managed infra. Alizé should position as the layer ABOVE that — deployment design, workflow architecture, governance, and outcome ownership. The infrastructure is commoditizing; the expertise isn't.

2. **Adopt Agent Teams / Subagents decision framework:** Document when to use parallel independent-context agents (Agent Teams) vs sequential shared-session agents (Subagents). Make this a standard Alizé deliverable — clients should understand which pattern fits their workflow before deployment begins.

3. **Build the human-in-the-loop default:** Agentova's SmartDrafts pattern (AI drafts, human approves) should be standard Alizé practice for regulated industries and any client with compliance requirements. Design every workflow with an approval gate.

4. **Multi-model routing as standard architecture:** OpenClaude's agent routing validates the approach. Alizé should design workflows that dynamically route to the most cost-effective model for each task type. Not Claude for everything — the right model for each job.

5. **Rakuten's one-week deployment pace as Alizé's target:** If Rakuten can deploy one enterprise agent per week per department, Alizé should promise similar timelines for standard workflows. Speed of deployment is a competitive advantage — market against consultants who take months.

6. **French SMB playbook against Agentova:** Alizé's direct competitor for small French businesses is Agentova. Differentiation points: managed service (not self-serve), French data residency, enterprise governance, multi-model routing. Target Agentova's unserved segment: businesses that want AI agents but don't want to build them themselves.

7. **AI bubble risk watch:** Chinese market observers flagging AI bubble. Monitor for signals of market correction — could create opportunity (cheaper inference) or risk (client pullback). Alizé should maintain lean operations so a correction doesn't threaten viability.
