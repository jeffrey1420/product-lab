## 1. Microsoft Agent Governance Toolkit — Open-Source Runtime Security for AI Agents

**Source:** [Microsoft Open Source Blog](https://opensource.microsoft.com/blog/2026/04/02/introducing-the-agent-governance-toolkit-open-source-runtime-security-for-ai-agents/) (April 2, 2026)

Microsoft released the **Agent Governance Toolkit** under MIT license — a 7-package monorepo addressing all 10 OWASP Agentic AI Top 10 risks with deterministic, sub-millisecond policy enforcement. Key facts:

- **Frameworks covered:** LangChain, AutoGen, CrewAI, Microsoft Agent Framework, Azure AI Foundry, Dify, LlamaIndex, OpenAI Agents SDK, Haystack, LangGraph, PydanticAI
- **Languages:** Python, TypeScript, Rust, Go, .NET
- **OWASP risks addressed:** goal hijacking, tool misuse, identity abuse, supply chain risks, code execution, memory poisoning, insecure inter-agent comms, cascading failures, human-agent trust exploitation, rogue agents
- **Key innovations:**
  - Agent OS: stateless policy engine intercepting every agent action before execution
  - DID-based identity with behavioral trust scoring
  - Cross-Model Verification Kernel (CMVK) with majority voting against memory poisoning
  - Inter-Agent Trust Protocol (IATP) encryption
  - Circuit breakers and SLO enforcement for cascading failures
  - Execution rings with resource limits
  - Automated kill switch for rogue agents
- **Microsoft's goal:** Move the project to a foundation (e.g., OWASP) for community stewardship — deliberately not vendor-controlled
- **Install:** `pip install agent-governance-toolkit[full]`
- **SLSA build provenance, 9,500+ tests, CodeQL, fuzzing via ClusterFuzzLite**

**What this means for Alizé:**
- Governance tooling is now commoditizing fast — Microsoft just made "we handle security" table stakes
- Alizé's differentiator must move up the stack: not just "we use governance tools" but "we design the workflow and own the outcomes"
- The CMVK (Cross-Model Verification Kernel) is particularly relevant: Alizé's multi-model architecture can use CMVK to verify outputs across Mistral/Claude/GPT rather than relying on single-model trust
- Microsoft open-sourcing governance = Alizé can embed this in client deployments at no cost — competitive moat shifts to consulting + workflow design
- The OWASP alignment is important: French enterprise clients will increasingly require OWASP compliance documentation for AI agent deployments
- Alizé should evaluate integrating this toolkit into its standard deployment package

## 6. Enterprise Agentic AI Landscape 2026 — Trust vs Lock-in Framework

**Source:** [Kai Waehner Blog](https://www.kai-waehner.de/blog/2026/04/06/enterprise-agentic-ai-landscape-2026-trust-flexibility-and-vendor-lock-in/) (April 6, 2026)

Independent practitioner analysis mapping 15+ vendors across two axes: **enterprise trust** and **vendor lock-in**. Key vendor positions:
- **High trust, low lock-in (Anthropic, OpenAI):** Strong safety culture, but API-based deployment = some lock-in at orchestration layer
- **High trust, high lock-in (Mistral, Aleph Alpha):** European sovereignty, but tighter integration requirements
- **Low trust, high lock-in (SAP Joule, Microsoft Copilot, Salesforce Einstein):** Convenient integration, but proprietary AI layer
- **Low trust, low lock-in (open-source models):** Maximum flexibility, but governance burden on customer

Key insight from the analysis: **"For regulated industries, a smaller and more trustworthy vendor may be the right choice."** The EU AI Act (high-risk obligations take effect August 2026) and Colorado AI Act (June 2026) will make trust documentation a procurement requirement.

**What this means for Alizé:**
- Alizé's positioning lands in "high trust, managed flexibility" — not the cheapest, not the most locked-in
- The EU AI Act enforcement timeline (August 2026) creates a window: French enterprises will need AI governance documentation before end of 2026
- Alizé should position itself as the "EU AI Act-ready AI agent partner" — helping clients build the compliance documentation (risk assessments, human oversight records, logging) that the Act requires
- Mistral positioning: European trust + acceptable lock-in for French regulated industries (finance, healthcare, public sector)
