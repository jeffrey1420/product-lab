## 1. MiniMax M2.7 Open Source — Self-Evolving Agent (April 12)

MiniMax released M2.7 as an open-source self-evolving agent model on April 12 — just days after being publicly named by Anthropic as a Chinese AI lab illicitly extracting capabilities from US models.

Key technical details:
- **Scores:** 56.22% SWE-Pro, 57.0% Terminal Bench 2, 66.6% medal rate across three trials

**The paradox:** MiniMax was named an IP adversary by Anthropic on April 7. Five days later they release a powerful open-source agent. ## 2. Google AI Vantage — Measuring Durable Skills in LLM Agents

Google Research published Vantage (April 13) — an LLM-based protocol for measuring collaboration, creativity, and critical thinking in LLM agents. Uses an **Executive LLM architecture** that orchestrates multiple AI personas in assessment conversations, actively steering scenarios to elicit evidence of specific skills.

Key findings:
- **Executive LLM outperforms Independent Agents** — coordinated multi-agent steering produces significantly higher evidence rates than independent agent spawning
- **Collaboration evidence rates:** 92.4% for Project Management, 85% for Conflict Resolution (Executive LLM condition)
- **Creativity scoring:** 0.88 Pearson correlation with human expert raters
- **Inter-rater agreement with humans:** Cohen's κ = 0.45–0.64 — comparable to inter-human agreement
- **Model used:** Gemini 2.5 Pro as the Executive LLM; Gemini 3 for creativity/critical thinking modules
- **Validation technique:** LLM-based simulation of human participants at known skill levels to de-risk assessment design before expensive human data collection

**Why this matters for Alize:** Vantage signals that Google is building the infrastructure to evaluate AI agents as workers — not as tools. If enterprise procurement evolves to require standardized "workplace performance" benchmarks for AI agents (similar to employee assessment rubrics), Alize needs to be able to demonstrate compliance. This is also a technical preview of how multi-agent coordination can be steered programmatically — relevant to Alize's agent orchestration layer.

## Deep Dive: MiniMax M2.7 + IP War — The Paradox Explained

This deserves its own analysis because it's the most strategically complex signal of this cycle.

**The timeline:**
- April 7: Anthropic names MiniMax, DeepSeek, and Moonshot as Chinese AI labs using distillation at scale to extract capabilities from US models. MiniMax API access to Claude is blocked.
- April 12: MiniMax releases M2.7 as open-source (Apache 2.0 or similar), matching Gemini-3.1 performance on agent benchmarks.

**Why release open-source after being named an adversary?**

1. **Preemptive legitimacy:** By releasing a strong open-source model, MiniMax creates millions of developers and companies with a vested interest in their survival. Sanctioning MiniMax now means disrupting those users.
2. **Community leverage:** Open-source adoption creates political cover — it's harder for the US to sanction a model that the global developer community depends on.
3. **Talent pipeline:** Open-source is also a recruitment vehicle. Developers who build on M2.7 are潜在的 employees or partners.
4. **Regulatory ambiguity:** The IP war is between governments. Open-source release creates ambiguity about whether using M2.7 (which is now globally available) constitutes using a "sanctioned Chinese AI product."

**The risk for Alize:** If Louis continues using MiniMax API while MiniMax is on the US adversary list, and if EU AI Act enforcement tightens in response to US-China tensions, Alize could face compliance issues. A French company using a US-designated adversary's API for client workflows is a governance red flag.

**Recommended action for Louis:** Migrate to Mistral API. Mistral is French, EU-hosted, and has no IP war exposure. The economics of Mistral are also competitive now — Mistral Small 4 (March 16) is a strong all-around model. For French enterprise clients, "powered by Mistral, hosted in France" is a better story anyway.
