## Signal 1: Shadow AI — The Enterprise Governance Crisis Alize Must Solve

The most operationally relevant story of the week: Shadow AI has become a mainstream enterprise security problem, and the market response is defining the next major buying criteria for B2B AI.

**The scale of the problem (Microsoft Security 2026 Data Security Index + Harmonic Security):**
- 32% of data security incidents now involve generative AI tools
- 22% of files sent to AI services contain sensitive information — including code, financial data, and M&A operational data
- 58% of employees use personal AI accounts at work (BYOAI phenomenon), up 5 points YoY
- 70% of workers use their own AI tools without IT validation
- Only 47% of organizations have deployed specific AI governance controls

**The concrete incident:** Bechtle Comsoft (Microsoft partner) detected a developer submitting production system architecture source code to both ChatGPT and Copilot — intent was legitimate (accelerate dev work), but the risk was real.

**The enterprise response — Microsoft Purview DSPM:**
Microsoft has built Data Security Posture Management directly into Purview, positioned as the answer to Shadow AI. Key capabilities:
- Real-time visibility into which AI tools are used, by whom, and what data is submitted
- Prompt inspection and sensitive information type (SIT) detection
- Automatic blocking of non-compliant data transfers to AI services
- 320+ predefined models + custom business rules

Key enterprise stat: 86% of security leaders now prefer integrated platforms to reduce alert fatigue — fragmented AI tool sprawl is becoming a liability.

**Key stat from the article:** 88% of decision-makers plan to increase cybersecurity budgets next year specifically to address AI-related risks.

**Alize angle — this is the validation and the opportunity:**
1. **Shadow AI proves the problem exists at scale.** Employees are already using AI tools with company data — often without governance. Alize's managed service model, where we deploy and govern AI agents inside the company's perimeter, directly addresses this.
2. **The 47% controls gap.** Most companies don't have AI-specific governance yet. Alize can position as the "governance-first AI operations partner" — we handle the agents AND the compliance framework.
3. **BYOAI + personal accounts = data leakage risk.** Alize's subscription model means the company uses our managed agents, not random consumer AI tools with corporate data. This is a genuine security selling point.
4. **The 22% of AI-submitted files contain M&A data.** This is the exact use case Alize targets (back-office ops for companies in M&A processes). The data confirms that sensitive operational data is already flowing to AI tools — the question is whether it's governed or not.

**Competitive differentiation angle:** While Anthropic, OpenAI, and Microsoft build the AI tools, someone has to govern them inside French PME. That's Alize's white space.

## Signal 2: Decathlon Production LLM Pipeline — Claude + Nova Multi-Model in the Wild

Decathlon published details on its Image Metadata Generation system, in production since end of 2025. This is one of the largest French enterprise LLM deployments publicly documented.

**Scale:** 2 million product images processed. Near-20 million total assets (communication, brand identity, product photos).

**Architecture decisions:**
- Multi-model: Claude (Anthropic) + Nova (Amazon) — used in parallel, not sequentially
- Batch async inference (not real-time) — treating AI inference as a job queue rather than synchronous API calls
- Images resized to 300px height/width (JPEG 85%) — reduces token consumption by ~96% vs original
- Results stored back to DAM (Digital Asset Management)
- Single prompt with two variants: packshots (product) vs contextual sports photography

**Why this matters for Alize:**
- Multi-model routing is validated in production at a major French enterprise. This confirms that companies are not standardizing on a single AI provider — they use multiple models for different tasks.
- The async batch processing pattern (jobs, not real-time) is directly relevant to how Alize would deploy agents for back-office ops — agents working on tasks in background, not requiring real-time human-in-the-loop.
- Decathlon's AWS quota constraints forced architectural optimization — a reminder that infrastructure constraints drive agent design decisions, not just model capabilities.
- The 96% token reduction from image resizing is a useful reference point for cost optimization in agentic workflows.

**French enterprise AI maturity signal:** A company the size of Decathlon (major French employer, significant digital transformation investment) is still very much in the "careful, batch-oriented, cost-conscious" phase of AI adoption. This is not AGI science fiction — it's pragmatic, industrial AI at scale. Alize's positioning (operational, governed, cost-predictable) maps directly to this reality.
