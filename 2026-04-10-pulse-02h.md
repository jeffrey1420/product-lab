## 1. Deloitte: SaaS Meets AI Agents — Pricing Will Never Be the Same

**Source:** [Deloitte 2026 SaaS AI Predictions](https://www.deloitte.com/us/en/insights/industry/technology/technology-media-and-telecom-predictions/2026/saas-ai-agents.html)

Deloitte's 2026 analysis of SaaS + AI agents has three major implications for Alizé:

### Outcome-Based Pricing Is Coming (Slowly)

The traditional per-seat, per-subscription model is under pressure. AI agents make it possible to measure *outcomes* — resolved support tickets, qualified leads, processed invoices — rather than just access. Deloitte predicts a shift toward value-based or outcome-based pricing for AI-augmented SaaS.

**What this means for Alizé:**
- Alizé is already positioned as a managed service, not a self-serve tool. This is the right bet.
- Monthly flat pricing with clear SLA is the right model (as noted in the Marie-Pierre persona from last pulse).
- Outcome-based pricing would be hard to implement early (attribution is complex), but the framing of "we guarantee results, not access" is a powerful positioning.
- Risk: if Alizé can't measure outcomes precisely, competitors who can will win on transparency.

### The "Attention Layer" Is a New Battleground

Deloitte identifies a new architectural layer emerging: the *interaction/attention layer* — the control center where humans supervise AI agents across multiple vendors and internal systems.

They identify three options for where this layer lives:
1. **Inside stand-alone SaaS apps** (Salesforce, ServiceNow keeping users in their ecosystem)
2. **A separate management platform** (third-party or vendor-provided)
3. **Agent marketplaces** (dynamic discovery and integration of new capabilities)

**What this means for Alizé:**
- Alizé's managed service offering IS the "control center" for PME that don't want to build it themselves.
- The "attention layer" is Alizé's value proposition: we own the complexity so the customer doesn't have to.
- Risk: Big Tech (Microsoft Copilot, Salesforce AgentForce) will compete in this space by bundling agent management into existing enterprise licenses. Alizé needs to be the trusted French alternative — local, human, accountable.

### The Interface Becomes Diagnostic and Conversational

AI agent interfaces in 2026 will be:
- **Personalized and proactive** — adapts to the user, suggests specific actions
- **Conversational** — natural language instead of menus and clicks. "Do this" not "click here"
- **Diagnostic** — transparency, explainability, reversibility, auditability are trust requirements

**What this means for Alizé:**
- Every client touchpoint (reports, dashboards, change notifications) needs to be explainable and human-readable.
- The EU AI Act's audit trail requirement is also a product requirement: if Alizé can show a clear decision log for every agent action, that's a compliance + UX differentiator.
- Alizé should lean into French-language conversational interfaces for its French-market clients.

## 2. Microsoft MCP Center — Enterprise MCP Registry Launches

**Source:** [mcp.microsoft.com](https://mcp.microsoft.com/)

Microsoft has launched an **MCP Center** — a registry and management platform for enterprise MCP (Model Context Protocol) servers. Key features:

- **Enterprise MCP Registry**: Organizations can register and manage their own MCP servers
- **Built-in integrations** across Microsoft's AI development environments (presumably Visual Studio Code, Copilot, etc.)
- **Vercel MCP**: Already integrated, can fetch project data, deployment logs, inspect failed deployments from within an AI client

**What this means for Alizé:**

MCP is rapidly becoming the standard for AI tool interoperability. Microsoft backing it validates the protocol. For Alizé:

1. **MCP as an integration layer**: Alizé should build its agent integrations via MCP wherever possible — it future-proofs against model/provider changes.
2. **OVHcloud + MCP**: If Alizé deploys on OVHcloud (French data sovereignty), MCP can connect to OVH's infrastructure APIs. This is a concrete EU AI Act / French data differentiator.
3. **Microsoft ecosystem risk**: If Microsoft makes Copilot Studio the "MCP control plane" for enterprise, Alizé needs a clear answer for French companies that want to stay out of Microsoft's orbit.

**Implication:** Alizé's technical architecture should standardize on MCP for all external integrations. This is both a product decision and a go-to-market differentiator (Alizé = French MCP integrator for regulated industries).

## 3. Agentova Competitive Intelligence — Deep Dive

**Source:** [agentova.ai](https://agentova.ai/), [Formaliv review](https://www.formalive.fr/avis-agentova/)

### What Agentova Is

Agentova is a **French no-code multi-agent platform** targeting entrepreneurs, TPE, and small PME. They have 8 pre-built AI agents with specific roles:

| Agent | Role |
|-------|------|
| Elisa | CEO: supervision, daily recaps, priority setting |
| Charlotte | Creative: social media content creation and planning |
| Amandine | SEO: keyword optimization, SEO content |
| Arthur | Sales: lead prospecting, qualification, pipeline |
| Ethan | Media Buyer: ad campaign management |
| Margot | Customer Support: 24/7 response, ticket filtering |
| Jenny | Admin: email management, task organization |
| Lucas | Data: analysis, reporting, dashboards |

### Scale and Traction

- **185+ entrepreneurs** on Trustpilot (4.2/5 score, 43 reviews)
- **+620 reviews** on their own site (likely aggregate across platforms)
- **3,000+ integrations** connected to the 8 agents
- **40,000+ estimated users** (based on Celesty case study with 40 employees using it)
- **Changelog updates**: Video generation AI (Jan 2025), phone agent Lily (Jan 2025), automatic prospect triggers (Feb 2025), 3000+ integrations (Feb 2025)

### Pricing Model

- Free 7-day trial
- Multiple plans based on number of agents + tasks
- No public pricing (opaque, likely €50–€500/month range based on similar tools)

### Case Studies

- **Celesty** (40 employees): "€40,000 monthly savings, doubled productivity" — strong claim
- **Mandyben Formation** (Marseille): +35% annual revenue, +6 employees
- **Closer University**: 2,000 members, x3 conversion

### How Alizé Compares

| Dimension | Agentova | Alizé |
|----------|----------|--------|
| Target | Entrepreneurs solo, TPE | PME/ETI, French market |
| Model | No-code self-serve platform | Managed service (we deploy + operate) |
| Agents | 8 pre-built generic agents | Custom-built per client workflow |
| Integrations | 3000+ (generic API integrations) | Deep integration into client's actual stack |
| Compliance | Not mentioned | EU AI Act compliance by design |
| Data sovereignty | Not mentioned | French/EU hosting |
| Support | Platform support | Named contact, human oversight |
| Pricing | Per-agent/task | Monthly flat retainer |
| Decision-maker | Solo entrepreneur | Gérant / Dirigeant |

**Key strategic insight:** Agentova is winning the *entrepreneur/TPE* segment with a self-serve product. Alizé should explicitly avoid competing here. Alizé's territory is the *smaller PME with 10–50 employees* that:
- Doesn't want to build/configure agents themselves
- Needs compliance documentation for EU AI Act
- Wants a French-speaking human to call when something breaks
- Has business processes complex enough that generic agents don't fit

This is a clear separation: Agentova = DIY for solopreneurs. Alizé = done-for-you for growing PME.

## 4. OpenAI Codex Desktop — "Agent Command Center"

**Source:** Zhihu discussion of [OpenAI Codex App](https://www.zhihu.com/question/2001978391264502048), February 2026

OpenAI released **Codex App** (macOS, February 2026), positioned as a **"Command Center for Agents"** — a desktop application that gives OpenAI's coding agents a persistent, controllable interface.

Key characterization from reports:
- Desktop agent interface with persistent memory of your codebase
- Direct access to OpenAI's most capable models
- Built for developers who want to orchestrate AI agents from a single interface

**What this means for Alizé:**

This is a developer/technical tool, not an enterprise operations tool. Codex App is relevant to Alizé only insofar as it shows that:
1. The "agent command center" concept is mainstreaming (see Deloitte's attention layer prediction)
2. OpenAI is competing directly with Anthropic's Claude Code and GitHub Copilot for developer mindshare
3. The AI agent tooling wars are accelerating — which means AI agents will become cheaper and more capable faster, compressing Alizé's time window to establish itself before Big Tech bundles these capabilities into enterprise agreements

**Alizé's response:** Don't compete on tooling — compete on *deployment expertise and trust*. Big Tech makes the models; Alizé makes the AI useful in a French regulated business context.

## Strategic Synthesis for Alizé

### New Findings to Integrate

1. **Outcome framing**: Shift Alizé's messaging from "we provide AI agents" to "we guarantee operational outcomes." Deloitte confirms this is where the market is heading. Even if we can't do pure outcome-based pricing yet, the framing aligns with what enterprise buyers will expect.

2. **MCP as architecture**: Alizé's integration layer should be MCP-first. This is now validated by Microsoft (MCP Center), reduces vendor lock-in, and is a legitimate technical differentiator for French IT decision-makers who need to explain their AI stack to auditors.

3. **EU AI Act = product feature**: The Deloitte article makes clear that explainability, audit trails, and compliance documentation are becoming UX requirements, not just legal requirements. Alizé should build a "compliance dashboard" that shows every agent decision, the data used, and the human validation step — this becomes a sales asset.

4. **Agentova is not Alizé's competitor**: Agentova targets solopreneurs with a self-serve platform. Alizé's actual competitive set is either (a) French mid-market companies doing DIY with OpenAI APIs, or (b) global managed service providers (Accenture, BCS, Deloitte's AI practices) that are too expensive for PME. Alizé fills the gap.
