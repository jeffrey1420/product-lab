## Signal 1: GPT-5 Released — OpenAI Moves Explicitly Into Back-Office Agent Territory

**Most significant development this cycle.**

OpenAI has released GPT-5 and the enterprise feature set is directly relevant to Alizé's positioning. Key capabilities announced (French OpenAI page, April 2026):

**Enterprise integrations:**
- Gmail and Google Calendar connected natively to GPT-5
- Company files from Google Drive and SharePoint used as context
- Respects existing permission structures

**What this means for Alizé:**
OpenAI is now explicitly targeting back-office operational automation — the exact market Alizé is positioning for. GPT-5 with Gmail/Calendar integration can already:
- Draft email responses based on inbox context
- Schedule meetings and coordinate across calendar
- Pull context from company documents
- Execute multi-step workflows across Google Workspace

**The threat:** A French PME decision-maker now has a plausible path to "AI agent for back-office ops" via GPT-5 + Google Workspace — without needing a specialized vertical agent provider like Alizé.

**Alizé's counter-positioning must be stronger:**
1. **Vertical depth** — GPT-5 is generic. Alizé should focus on industry-specific workflows (insurance claims follow-up, invoice processing, regulatory compliance for French companies)
2. **Governance layer** — GPT-5 lacks the deterministic guardrails that regulated industries (finance, insurance, healthcare) require. Alizé's "Context Graph"-style governance is the differentiator
3. **Sovereignty angle** — GPT-5 is US-infrastructure. French companies in regulated sectors (banques, assurances, santé) have data residency requirements that GPT-5 cannot easily satisfy. Mistral AI + French infrastructure = the Alizé moat
4. **Speed to deployment** — OpenAI's enterprise sales cycle is long (3-6 months). Alizé can move faster on specific use cases

**Alizé must move to domain acquisition and first pilot NOW while GPT-5 is still being evaluated by enterprise IT departments, not after.**

## Signal 2: Mistral Voxtral TTS — Open-Weight Voice Agents Now Possible

**Announced March 23, 2026.**

Mistral AI released Voxtral TTS — an open-weight text-to-speech model under Apache 2.0 license. Key specs:
- 4B parameters
- Zero-shot voice cloning from 3 seconds of audio
- Natural, emotionally expressive speech
- Designed for voice agent applications

The Voxtral family (transcription + TTS) is now fully open-weight. This matters for Alizé because:

1. **Voice-first agent modality** becomes feasible without paying ElevenLabs or proprietary API fees
2. **French-language voice agents** are now technically achievable with open tools — a打电话给客户 follow-up agent, voice-based appointment booking, voice-synthesized report delivery
3. **Alizé voice agent roadmap** — this validates that voice is a credible future modality for Alizé agents. A French voice agent that calls clients, leaves voicemails, handles voice-based appointment booking would be differentiated
4. **Cost structure** — open-weight means no per-minute TTS costs, which makes voice agents economically viable at scale

**Action item for Louis/Gabin:** Evaluate Voxtral TTS as part of the Alizé technical stack. Even if voice isn't in the MVP, the architecture should be voice-ready.
