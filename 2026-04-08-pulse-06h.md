## Signal 1: Claude Mythos + Project Glasswing — Anthropic's Most Capable Model Is Too Dangerous to Release

Anthropic officially announced **Claude Mythos** (a fourth, previously unannounced tier codenamed "Copybara") on April 7, 2026. Mythos is described as superior to any existing frontier model — Opus included — with the highest scores ever recorded on software coding tasks. The model has already found thousands of zero-day vulnerabilities during internal testing, including a 27-year-old bug in OpenBSD and a 16-year-old video software flaw that evaded five million automated test hits.

Because of its offensive cybersecurity capabilities, Mythos is NOT being released publicly. Instead, it powers **Project Glasswing**, announced simultaneously. Glasswing is a defensive coalition that includes Amazon, Anthropic, Apple, Broadcom, Cisco, CrowdStrike, the Linux Foundation, Microsoft, and Palo Alto Networks — organized to secure the world's critical software before Mythos-class capabilities proliferate among malicious actors.

The timing is not accidental: in September 2025, a Chinese state-sponsored group used Claude Code's agentic capabilities to execute what Anthropic described as "the first reported AI-orchestrated cyber espionage campaign." Anthropic detected it and published the findings. The Glasswing coalition is the direct institutional response.

**What this means for Alizé:**
Two competing narratives are emerging simultaneously. Anthropic is arguing that agentic AI is so powerful that even the most capable lab won't release the model publicly — while simultaneously building the world's most aggressive enterprise AI integration (M365 connectors, Windows PC control). The Glasswing coalition — which includes Microsoft, Cisco, CrowdStrike — signals that the major enterprise security vendors see agentic AI as both an infrastructure standard and a threat requiring coordinated defense. For Alizé, this is validation of the governance + MCP connector thesis: the product isn't the agent, it's the layer that makes agents safe to deploy in regulated French PME.

## Signal 2: Claude Code Source Leak + Critical Permission Bypass — Two Weeks of Bad Press for Anthropic

On March 31, 2026, Anthropic accidentally published a debugging JavaScript sourcemap for Claude Code v2.1.88 to npm. Within hours, two University of British Columbia students reconstructed the source: 512,000 lines of TypeScript across 1,900 files. The leak spread globally and now persists on the internet.

Then, Adversa AI discovered a critical vulnerability in Claude Code's permission system. The issue: Claude Code caps subcommand analysis at 50, with a fallback to "ask" for anything beyond. The security assumption was that 50 is "generous" — legitimate commands don't split that wide. But prompt injection via a malicious CLAUDE.md file can trigger 50+ compound commands that look like a routine build process. When the cap is exceeded, ALL deny rules, security validators, and command injection detection are silently skipped — and the user gets no indication that the security boundary has been bypassed.

Impact: credential theft (SSH private keys, AWS tokens, GitHub tokens, npm tokens), supply chain compromise, CI/CD pipeline poisoning. The LLM safety layer caught some obviously malicious payloads (defense-in-depth), but the permission system bug is independent of that layer.

The timeline matters: source leak March 31, vulnerability discovered April 2, Mexican government cyberattack using weaponized Claude Code reported shortly after.

**What this means for Alizé:**
The Claude Code vulnerability is a concrete, reproducible example of the governance gap. An enterprise deploying Claude Code agents has a permission system they believe is a security boundary — but it can be silently bypassed by a malicious repository file. This is exactly the class of risk that Alizé's managed service model is meant to address: not just deploying agents, but auditing, monitoring, and securing the agent's operational environment. The fact that this vulnerability exists in a product built by the world's most safety-conscious AI lab proves that even frontier-grade tooling requires third-party governance.

Two consecutive days of major outages, impacting the same core functionality (login, voice, chat), is a pattern that enterprise IT teams will notice. Combined with the consecutive days of bad security news (vulnerability April 2, Mythos announcement April 7), April 2026 has been a difficult week for Anthropic's enterprise reputation.

**What this means for Alizé:**
Alizé's managed service positioning is reinforced by this signal. If a French PME is running critical back-office automation on a direct Claude API integration and the service goes down for two consecutive days, that's a business continuity problem. Alizé's managed service model — with fallback protocols, SLA commitments, and operational oversight — is the solution. The 40% of agentic AI projects at risk of cancellation (from last pulse's KPMG data) are often at risk because of these kinds of operational unknowns. A managed service that includes reliability guarantees is a different product than a raw API integration.

## Signal 5: Google DeepMind Maps "AI Agent Traps" — Six Classes of Web Attacks on AI Agents

Google DeepMind published research identifying **six classes of attacks that malicious web content can mount against AI agents**:

1. **Content injection traps**: Hidden commands in HTML comments, metadata, JavaScript, or steganography that exploit the gap between human-visible rendering and machine-parsed content
2. **Semantic manipulation traps**: Carefully crafted language to trigger cognitive biases, bypass verification filters, or reshape the agent's personality
3. **Cognitive state traps**: Poisoning the agent's external data sources or corrupting long-term memory stores
4. **Behavioral control traps**: Jailbreaks embedded in external resources, forcing agents to leak privileged information or spawn compromised sub-agents
5. **Systemic traps**: Targeting multi-agent environments — exploiting homogeneity, behavior synchronization, and consensus processes across agent networks
6. **Human-in-the-loop traps**: Commandeering the agent to attack its own human operator (e.g., invisible prompt injections that make the agent repeat ransomware commands as "remediation instructions")

The research paper explicitly calls this "a foundational challenge" requiring sustained collaboration between developers, security researchers, and policymakers. Proposed defenses: model hardening, runtime defenses, content governance frameworks, and standardized evaluation benchmarks.

**What this means for Alizé:**
This research validates the agentic security surface that Alizé must address. The "systemic traps" class — attacks targeting multi-agent environments — is particularly relevant for Alizé's managed service model where multiple Claude agents might be running in the same client environment. The human-in-the-loop trap is a direct argument for Alizé's oversight + escalation protocol: if an agent can be turned against its operator, you need a governance layer that the agent cannot manipulate.
