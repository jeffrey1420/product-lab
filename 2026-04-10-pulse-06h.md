### Aembit IAM for Agentic AI — Now Generally Available

**Source:** [Aembit blog](https://aembit.io/blog/aembit-iam-for-agentic-ai-is-now-generally-available/)

Aembit has released its **Identity and Access Management for Agentic AI** as GA (Generally Available). This is directly relevant to Alizé's MCP-first architecture strategy.

**Two new MCP components:**

1. **MCP Authorization Server** (Aembit Cloud, managed):
   - Implements OAuth 2.1 authorization code flow per MCP spec
   - AI agents authenticate and receive access tokens minted by Aembit policies
   - Integrates with Okta, Azure AD, Google via OIDC/SAML
   - User context from human authentication flows into access decisions

2. **MCP Identity Gateway** (deployed as Linux VM in customer environment):
   - Validates tokens and enforces policy on every MCP request
   - Performs credential exchange on the agent's behalf — **agent never holds direct credentials**
   - Works for custom MCP servers and third-party SaaS MCP servers
   - Controls network boundaries and data locality

**Blended Identity — the key innovation:**
Aembit evaluates *both* the AI agent identity and the human operator identity in a single policy decision. The question: "Is this specific user, using this specific agent, authorized to access this specific resource right now?"

Examples:
- Engineers can use Claude Desktop to access Jira, but only the security team can use it to access the vulnerability scanner
- A user can access Confluence through Claude Desktop but not through an untrusted third-party agent
- One user's AI agent access can be revoked without affecting other users

**Pricing:**
- Free Starter tier (dev/POC)
- AI Teams: **$20 per agent per month**
- Enterprise: tier above that

**What this means for Alizé:**
- **MCP is now a full identity and access management battleground.** Aembit, Aurascape, and Microsoft's MCP Center are all building in this space.
- Alizé should evaluate whether to integrate Aembit or build similar Blended Identity capabilities into its managed service offering
- The $20/agent/month pricing gives Alizé a benchmark: if Aembit charges this for IAM alone, Alizé's full managed service should be priced significantly higher for French PME
- "Agent never holds direct credentials" is a powerful security narrative for EU AI Act compliance conversations

## 5. Enterprise Agentic AI Landscape 2026: The Trust Framework

**Source:** [Kai Waehner - Enterprise Agentic AI Landscape 2026](https://www.kai-waehner.de/blog/2026/04/06/enterprise-agentic-ai-landscape-2026-trust-flexibility-and-vendor-lock-in/)

This is an important analytical framework for understanding where Alizé fits in the competitive landscape. Two dimensions define enterprise AI vendor selection:

**Axis 1: Enterprise Trust** — commitment to responsible AI, data governance, GDPR/EU AI Act compliance, jurisdiction/s sovereignty

**Axis 2: Vendor Lock-in** — API dependency, agent framework capture, data gravity, ecosystem entanglement

**Key positions relevant to Alizé:**

| Vendor | Trust | Lock-in | Notes |
|--------|-------|---------|-------|
| Anthropic | High | High | Best safety culture, but proprietary + Claude Code creates lock-in |
| Mistral | High | Flexible | Open-weight, French jurisdiction, EU-aligned — **best fit for Alizé** |
| Google | Medium-High | High | Powerful but Captured (ecosystem) |
| Microsoft | Medium | Very High | Most Captured (Copilot + Azure + 365) |
| OpenAI | Medium | High | Powerful models but API lock-in + training data concerns |
| AWS | Medium | High | Bedrock partially addresses lock-in |
| Aleph Alpha | High | Flexible | German, EU-aligned, but smaller ecosystem |

**Mistral's position is the sweet spot for regulated European enterprises:** High trust + Flexible (open-weight, French jurisdiction, EU regulatory alignment). This validates Alizé's potential positioning as a Mistral-first managed service for French PME.
