## Signal 3: Syntaqlite — Solo Developer, 250h, AI Coding Agents, Production Devtools

The Hacker News front page today featured a detailed case study from Lalit Maganti (Google engineer) who spent ~250 hours over 3 months building [Syntaqlite](https://github.com/LalitMaganti/syntaqlite) — a production-quality SQLite devtools suite — using AI coding agents (Claude Code on Max plan).

**What happened:**
- First attempt (AI-maximalist approach): worked functionally but produced spaghetti codebase (several-thousand-line files, no clear architecture, fragile)
- Second attempt: human architect + AI for implementation + explicit separation of concerns → production quality
- Key insight: AI handles repetitive/structured work; human handles architecture and quality control
- 400+ grammar rules in SQLite, no formal spec, no stable parser API — hardest part required extracting C code from SQLite's dense source

**Lessons for Alizé:**
1. AI agents work best with clear human architectural direction
2. "Vibe coding" alone produces unmaintainable code — human oversight is mandatory, not optional
3. The most productive pattern: human sets scope and reviews; AI executes
4. Syntaqlite validates that 1 developer + AI can build production tools in months, not years

**Alizé implication:** This is the development speed reality check. Alizé's infrastructure can be built faster than traditional estimates suggest — but needs architectural discipline. The Syntaqlite post also provides a credible external reference that Louis can cite to prospects: "One developer built a production devtool suite in 3 months with AI. Your internal tools can be built the same way, with Alizé managing the agents."

## Signal 4: Enterprise AI Deployment Patterns — Customer Service at 60-75% Automation, Legal/Finance Following

From Trensee's enterprise AI deployment analysis (March 2026):

**Customer service:** 60-75% first-contact automation is achievable; remaining 25-40% still requires human agents. Key finding: targeting 100% automation drives sharp customer dissatisfaction — escalation logic design is the most critical deployment element.

**Software engineering:** 30-55% speed improvements on repetitive tasks; AI contribution to complex feature design and architecture remains limited.

**Legal/compliance:** AI agents effective in draft generation, issue screening, document search — but human review before final legal opinion is mandatory everywhere.

**Financial services:** Internal workflow automation first (fraud detection augmentation, compliance report drafting, credit application review); customer-facing deployment moves more cautiously due to regulation.

**Common failure patterns:**
1. Waiting for perfect AI (forfeits near-term value)
2. Tech deployment without change management
3. No ROI measurement framework
4. Underestimating data quality requirements

**Alizé implication:** These deployment patterns directly map to Alizé's target segments. French PME in these functions (customer service for e-commerce, compliance for regulated industries) are the buyers. The escalation logic / human oversight framing is exactly Alizé's governance positioning.
