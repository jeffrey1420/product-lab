## Signal 1: Claw Code — 72K GitHub Stars in First Days (April 2, 2026)

Claw Code launched as an independent open-source foundation for AI-assisted software development, achieving 72,000 GitHub stars and 72,600 forks within days of public release.

**What it is:**
- Open-source AI coding agent framework in Python and Rust
- Clean-room rewrite of AI coding harness patterns — no proprietary code or third-party weights
- Focuses specifically on the "harness layer": how context flows, how tools connect, how agent decisions get made and observed
- Addresses the gap between capable LLMs and the opaque proprietary control layers that orchestrate them

**Architecture:**
- Python src/ workspace for current development
- Rust port in progress (memory-safe production runtime target)
- CLI utilities: summaries, manifests, subsystem inspection, parity audits
- Built using AI-assisted development itself (oh-my-codex workflow)

**Why it matters for Alizé:**
- The "harness layer" is exactly what Alizé would build for managed AI agents for French PME — the orchestration, context management, and observability layer between LLMs and business workflows
- Claw Code validates that open-source harness infrastructure is a real market category, not just internal tooling
- The fact that it hit 72K stars in days shows massive developer appetite for alternatives to proprietary agent frameworks
- **Alizé implication:** Alizé's managed service positioning is reinforced — companies need help building the harness, not just accessing the LLM. The real value is in deployment, governance, and operation.

## Signal 2: Microsoft Agent Framework — 1.0.0 Production Stable (April 2, 2026)

Microsoft released its Agent Framework 1.0.0 for both Python and .NET, promoting it from release candidate to Production/Stable status.

**What's new:**
- Python: agent-framework 1.0.0, agent-framework-core 1.0.0, agent-framework-openai 1.0.0, agent-framework-foundry 1.0.0
- .NET: Microsoft.Agents.AI.Foundry replacing Azure.AI.Foundry (naming consolidation)
- Breaking changes: Message construction now uses Message(contents=[...]) instead of Message(text=...)
- Handoff Orchestrations marked as Experimental
- DevUI: accepts legacy payload formats (text, message, content, input, data) for backward compatibility

**Enterprise angle:**
- Microsoft Foundry integration is now central (renamed from Azure AI Foundry)
- Multi-agent orchestration is a first-class concern
- Checkpoint/resumption for agent workflows is stable
- Telemetry and observability are built in

**Alizé implication:** Microsoft's push to 1.0.0 signals that enterprise agent frameworks are crossing the maturity threshold. Alizé should consider Microsoft Agent Framework as a potential building block or reference architecture for the Alizé orchestration layer. The naming shift toward "Foundry" (vs "Azure AI") also signals a more platform-agnostic positioning.
